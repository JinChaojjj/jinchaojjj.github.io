  • 修改根目录下的gradle文件夹里的wrapper文件夹里的gradle-wrapper.properties
distributionBase=GRADLE_USER_HOME

#修改distributionUrl为你的压缩包所在地址
distributionUrl=file:///d:/gradle-5.6.4-bin.zip
zipStoreBase=GRADLE_USER_HOME

  • 修改根目录下的build.gradle文件
configure(allprojects) { project ->
    dependencyManagement {
        repositories {
            //新增以下2个阿里云镜像

2.2 问题

编译spring源码其实整体的结构并不复杂，但是网上很多教程并没有把这个东西讲清楚

可以理解为spring源码是一个用gradle构建的项目

我们将源码下载到本地之后，通过gradle进行项目的构建，那么构建项目的目的是什么呢？

—就是为了将源代码进行编译，构造一个jar包！maven工具的本质不就是帮助我们下载jar包嘛

同样，编译得到jar包和maven导入jar包都是一样的，我们编译spring源码只要得到jar包，并且没有语法上的错误基本上就是编译成功了，至于其他的错误，有些是git的问题，有些是扫描结构的问题，其实咱们可以忽略

那么那么那么问题来了生成了怎么用呢，对吧，这方法就很多了，现在启动的这个项目主要是编译源代码使用的，当然你也可以更改源代码，在自己需要的地方进行参数的打印、更改逻辑等都是可以的，并且更改代码之后重新编译就好了，这样其他项目引用的这个jar包就是你更改之后的jar

这也是我们编译源代码的最主要的目的！！！

话说回来，怎么用呢

因为jar生成之后就在我们的硬盘中，1.你可以使用maven导入spring然后讲关联的jar包地址改了

2.直接从硬盘中导入使用

不要去动jar的位置就好了，jar就放在那，这样你重新编译之后还是在原位的，不需要你重新进行项目的配置。！！
